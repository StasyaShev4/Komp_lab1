## Разработка грамматики
```
1) <Struct> → struct <NameStruct>
2) <NameStruct> → <Letter> <NameStruct> | "{" <Fields>
3) <Fields> → string <Fild> | int <Fild>  | bool <Fild>  | float <Fild>  | array <Fild> | "}" <Final> 
4) <Fild> → $<NameVariable>
5) <NameVariable> → <Letter><NameVariable> | ";" <Fields>
6) <Final> → ";"
7) <Letter> → "a" | "b" | ... | "z" | "A" | "B" | ... | "Z"
   
 - Z = ‹Struct›;
 - VT = {a, b, c, ..., z, A, B, C, ..., Z,";","{","}", }; 
 - VN = {<Strut>, <NameStruct>, <Letter>, <Fields>, <Field>,<NameVariable>, <Final>}.
```

## Классификация грамматики (по Хомскому)
Согласно классификации Хомского, грамматика `G[<Struct>]` является автоматной.
Правила относящиеся  к классу **праворекурсивных продукций** вида:
A → aB | a

```
1. ‹Struct› → struct ‹NameStruct›
2. ‹NameStruct› → "{" ‹Fields›
3. ‹Fields› → string ‹Fild›
4. ‹Fields› → int ‹Fild›
5. ‹Fields› → bool ‹Fild›
6. ‹Fields› → float ‹Fild›
7. ‹Fields› → array ‹Fild›
8. ‹Fields› → "}" ‹Final›
9. ‹Fild› → $‹NameVariable›
10. ‹NameVariable› → ";" ‹Fields›
11. ‹Final› → ";"
```

## Диагностика и нейтрализация синтаксических ошибок.
В данной работе используется алгоритм нейтрализации синтаксических ошибок методом Айронса.
Алгоритм реализован на основе конечного автомата с восстановлением состояния через функцию `Recover()`.
В случае возникновения синтаксической ошибки парсер переходит в состояние `ERROR`, после чего вызывается процедура восстановления.
В процессе восстановления парсер последовательно отбрасывает токены до тех пор, пока не встретит один из синхронизирующих символов, соответствующих грамматике описания структур.

В реализации используются следующие ключевые точки восстановления:
- ключевое слово `struct`
- типы данных: `string`, `int`, `float`, `bool`, `array`
- разделители: `;`, `}`
